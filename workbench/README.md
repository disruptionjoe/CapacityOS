# Workbench

This folder holds transitional material used while designing, pressure-testing,
or migrating into CapacityOS.

It is intentionally separate from the core engine surface.

Use `workbench/` for things like:

- staging plans
- migration maps
- import bundles
- temporary support material for cutover work

In this repo:

- `00-plan.md` is the current staging plan
- `migration-bundles/` holds the current JoeEA migration-support bundle set

This material may still be useful to builders, but it should not be mistaken
for the reusable engine itself.
