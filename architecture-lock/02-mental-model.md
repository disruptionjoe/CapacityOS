# 02 - Mental Model

## One-Sentence Explanation

CapacityOS is an engine that keeps domains coherent, keeps live work moving,
and surfaces what only Joe needs to decide or do next.

## Joe-Facing Model

Joe does not need to think in schemas, folders, or lifecycle internals.

Joe mainly needs to think in:

- domains
- goals
- the trusted queue
- review packages
- Joe-only actions
- shipped or blocked outcomes

## Core User Experience

The default daily surface is the daily operating view.

It should answer:

- what the system is already moving
- what the top agent-ready work is by domain and goal
- what only Joe needs to do now
- what review or consequence checkpoints need judgment

Backlog exists, but it is not the default surface.

The default surface should emphasize the trusted queue and Joe-only work, not
the full volume of speculative or exploratory items.

## Intake Model

Conversation is part of system intake.

Most of the time, when something becomes real work:

- it enters the intake loop
- it becomes structured state
- it may later become backlog, queue, or execution

The default assumption is `^que`, even when no notation is written.

`^now` exists, but only when immediate handling is explicitly requested.

## What Joe Sees Most Often

- the daily operating view
- high-priority Joe tasks
- review packages prepared by the system
- consequence steps that need human action
- issues worth noticing

## What Joe Should Rarely Need To See

- raw runtime state folders
- engine policy files
- cadence mechanics
- parser-facing structure details
- background coordination work that does not require judgment

## Recommended Vocabulary

### Joe-facing words

- `Domain` - a meaningful area of life or work with its own goals and rules
- `Queue` - the trusted set of work ready to move
- `Backlog` - potential work not yet promoted into the trusted queue
- `Review package` - a prepared object ready for human judgment
- `Joe task` - something only Joe can do now
- `Issue` - a structured signal that something should improve

### Internal words

- engine
- canon
- runtime
- intake receipt
- triage proposal
- backlog item
- queue item
- bundle
- decision record
- run

## Mental Boundaries

- domains provide coherence
- runtime shows what is happening now
- the engine provides reusable structure and cadence
- human checkpoints exist at high-leverage transitions, not on every step

## Success Check

If Joe opens CapacityOS, he should be able to infer:

- what domains matter now
- what the trusted queue is pushing forward
- what only he needs to do
- what the system reviewed or prepared for him
- where durable meaning ends and live operational state begins
